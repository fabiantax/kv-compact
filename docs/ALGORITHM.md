# KV Cache Compaction via Attention Matching — Algorithm Documentation

**Implementation of:** "Fast KV Compaction via Attention Matching" (Zweiger et al., 2026)
**Paper:** [arXiv:2602.16284](https://arxiv.org/abs/2602.16284)
**Authors:** Adam Zweiger, Xinghong Fu, Han Guo, Yoon Kim — MIT

---

## Overview

This library implements a training-free, post-hoc algorithm for compressing the KV cache of transformer-based LLMs. Instead of simply evicting tokens, it constructs a compact representation that preserves the *attention behavior* of the original cache — both the attention output (what information is retrieved) and the attention mass (how much weight old context receives vs. new tokens).

The algorithm compresses T cached tokens down to t tokens (where t ≪ T) by solving three sequential optimization problems, each with closed-form or convex solutions. No gradient descent through the model is needed, making it orders of magnitude faster than prior latent-space compression methods.

**Key result:** 50x compression in ~20 seconds with minimal accuracy loss.

---

## The Problem

Every token generated by an LLM requires attending over all previously cached key-value pairs. The KV cache grows linearly with context length and becomes a memory bottleneck for long sequences (60k+ tokens can consume multiple gigabytes per request).

**What we want:** Given a KV cache with T entries, produce a compact cache with t entries such that for any future query q:

```
Original:    y = softmax(q·K^T / √d_k) · V
Compacted:   ŷ = softmax(q·C_k^T / √d_k + β) · C_v    ≈ y
```

where C_k are selected keys, β are per-key attention biases, and C_v are refitted values.

### Why simple eviction fails

Token eviction (dropping low-attention tokens) works at low compression but degrades sharply beyond ~5x. The problem is fundamental: with fewer tokens, the total attention mass changes, distorting the balance between old context and new tokens during generation. No amount of clever token selection can fix this without modifying the key/value representations themselves.

## The Three-Step Pipeline

The joint optimization of (C_k, β, C_v) decomposes into three independent subproblems, solved sequentially. This decomposition is the key insight that makes the method both fast and mathematically clean.

### Step 1: Key Selection (Paper Section 3.1)

**Goal:** Choose which t of the T original keys to retain.

**Algorithm (Highest Attention Keys):**

1. Compute scaled attention scores between reference queries and all keys:
   ```
   S = Q_ref · K^T / √d_k       [n_q × T]
   ```

2. Convert to attention weights via softmax:
   ```
   A[i,j] = exp(S[i,j]) / Σ_k exp(S[i,k])
   ```

3. Score each key by its maximum attention weight across all queries:
   ```
   score[j] = max_i A[i,j]
   ```
   This ensures a key critical to *any* query pattern is retained.

4. Select the top-t keys by score, then re-sort by position for cache locality.

**Complexity:** O(n_q · T · d_k) for the matmul, O(T log t) for selection.

**Implementation:** `compact_head_highest_attn()` in `kv-compact-math.h`

### Step 2: Bias Fitting via NNLS (Paper Section 3.2, Equation 4)

**Goal:** Find bias vector β so the compacted cache preserves the original attention mass.

**Why biases are necessary:** Without biases, mass matching is impossible when t < T. Consider q = 0: original mass = T (sum of T ones), compacted mass = t. The bias exp(β_j) lets each compacted key represent the aggregate mass of multiple original keys.

**Formulation:**

The unnormalized attention mass for query i over the full cache is:
```
m_i = Σ_j exp(q_i · K_j^T / √d_k)
```

For the compacted cache with biases:
```
m̂_i = Σ_j exp(q_i · C_k_j^T / √d_k + β_j) = Σ_j M[i,j] · w_j
```

where w_j = exp(β_j) ≥ 0 and M[i,j] = exp(q_i · C_k_j^T / √d_k).

This yields the constrained linear system:
```
min_{w ≥ 0}  ‖M·w − m‖²
```

**Solver — Lawson-Hanson Active-Set NNLS:**

Our implementation uses the Lawson-Hanson active-set method rather than projected gradient descent. This classical algorithm maintains a partition of variables into an active set (forced to zero) and a passive set (free to vary):

1. Initialize all variables in the active set (w = 0).
2. Find the variable with the largest negative gradient in the active set.
3. Move it to the passive set and solve the unconstrained subproblem on passive variables.
4. If any passive variable goes negative, move the most violating one back to active.
5. Repeat until the KKT conditions are satisfied.

**Advantages over gradient descent:** Finite convergence guarantee, no step-size tuning, exact solution up to floating-point precision.

**Final conversion:** β_j = log(max(w_j, 10⁻¹²))

**Implementation:** `nnls_solve()` in `kv-compact-math.h`

### Step 3: Value Refitting via Least Squares (Paper Section 3.3, Equation 6)

**Goal:** Find replacement values C_v so the attention output with compacted keys + biases matches the original.

**Formulation:**

Compute the compacted attention weights (what a query would see with the compacted cache):
```
X[i,j] = softmax_j(S[i, selected[j]] + β_j)     [n_q × t]
```

Compute the original attention output (ground truth):
```
Y[i,:] = Σ_j A[i,j] · V[j,:]                     [n_q × d_v]
```

Solve the least-squares problem:
```
min_{C_v}  ‖X · C_v − Y‖²
```

**Solution via normal equations:**
```
C_v = (X^T·X + λI)⁻¹ · X^T · Y
```

Solved directly using Gaussian elimination with partial pivoting. The matrix X^T·X is small (t × t, typically 50–500), so the O(t³) direct solve is fast and exact.

**Ridge regularization (λ = 10⁻⁶):** Ensures invertibility and prevents extreme values when reference queries are nearly collinear.

**Implementation:** `ls_solve()` in `kv-compact-math.h`

## Extensions Beyond the Paper

### Lawson-Hanson NNLS Solver

The paper uses projected gradient descent for the NNLS problem. We replaced it with the Lawson-Hanson active-set method, which provides exact solutions with finite convergence guarantees. This eliminates the need to tune step sizes or iteration counts.

### Diversity-Aware Key Selection

Standard key selection can pick redundant keys that all attend to similar patterns. Our diversity extension applies a cosine-similarity penalty during selection:

```
adjusted_score[j] = score[j] × (1 − strength × max_similarity_to_selected[j])
```

This greedy approach selects keys that cover more of the query space. In testing, diversity selection improved cosine similarity from 0.998 to 0.9999 at 50% compression.

**Implementation:** `select_keys_diverse()` in `kv-compact-api.cpp`

### Shared Prefix Preservation

For multi-agent systems sharing a common prompt prefix, the first n_shared_prefix token positions are always retained (never evicted). This ensures the shared context remains intact across agents while only compacting agent-specific tokens.

### Cheap Q_ref from K-Vector Proxy

The paper generates reference queries via "repeat-prefill" — feeding the context through the model twice. This costs ~40% of total compaction time. As a cheaper alternative, we sample K vectors as proxy queries, exploiting the observation that K and Q live in similar subspaces.

Samples are quadratically biased toward recent tokens (more likely to be queried during generation). In testing, this achieves ~10x speedup (44ms vs 435ms) with equivalent quality at moderate compression.

**Implementation:** `generate_cheap_qref()` in `kv-compact-api.cpp`

### Iterative Refinement

After the initial 3-step pipeline, optional refinement rounds evaluate per-key reconstruction error, swap the worst selected keys with the best unused ones, then re-run Steps 2–3. Typically 2–3 rounds suffice.

### Multi-Round Compaction

For incrementally growing caches, `kv_compact_multi_round()` applies compaction repeatedly. Each round compacts the output of the previous round, with beta folded into K vectors and C_v replacing V. Selected indices map back to original positions.

### ROCm/HIP GPU Acceleration

The attention scoring matmul (Q_ref · K^T) is the computational bottleneck. For AMD APUs (targeting Ryzen AI 395 Pro Max with RDNA 3.5 integrated GPU), we provide HIP-accelerated tiled matrix multiply using shared memory. The APU's unified memory architecture eliminates explicit device-host copies.

**Implementation:** `kv-compact-hip.hip`

## Per-Head Sensitivity Weighting (Paper Section 4)

Not all attention heads are equally sensitive to compression. The paper demonstrates that relative head sensitivity rankings are stable across inputs, enabling precomputed per-head budget allocation.

When `use_sensitivity` is enabled, key scores are weighted by per-head sensitivity before the shared selection step:
```
weighted_score[j] += head_sensitivity[h] × head_score[h][j]
```

This allocates more of the budget to heads where compression hurts most.

**Implementation:** `compact_kv_cache()` in `kv-compact-math.h` (Section 3.4 multi-head path)

## Quality Metrics

The library computes three quality metrics after compaction:

| Metric | Formula | Interpretation |
|--------|---------|----------------|
| **Cosine Similarity** | (ŷ · y) / (‖ŷ‖ · ‖y‖) | Directional agreement (1.0 = perfect) |
| **MSE** | (1/d_v) Σ (ŷ_d − y_d)² | Per-dimension squared error |
| **Agreement Rate** | fraction with matching argmax | Top-1 prediction preservation |

**Observed quality (unit tests):**
- 2x compression: cosine similarity > 0.999
- 4x compression: MSE improvement over eviction ~4,000,000x
- Diversity selection: cosine similarity 0.998 → 0.9999

## Numerical Stability

| Technique | Purpose | Location |
|-----------|---------|----------|
| Max-shift in exp/softmax | Prevents overflow (exp(x) for x > 88) | `softmax_rows()`, `exp_rows_stable()` |
| Non-negativity floor (10⁻¹²) | Prevents log(0) = −∞ in beta | `nnls_solve()` |
| Partial pivoting | Minimizes rounding error growth | `ls_solve()` |
| Ridge regularization (10⁻⁶) | Bounds condition number of X^T·X | `ls_solve()` |
| Division guards (10⁻⁸ / 10⁻¹²) | Prevents NaN from zero division | Throughout |

## Memory Layout

**KV tensors:** `[T × n_head_kv × d]` row-major. Each row = one token, all heads concatenated.

**Result structure:**
```c
kv_compact_result {
    int     t;                  // tokens kept
    int   * selected_indices;   // [t] original positions (sorted)
    float ** beta;              // [n_head_kv][t] attention biases
    float ** C_v;               // [n_head_kv][t × d_v] refitted values
    kv_compact_stats stats;     // quality + timing metrics
}
```

## Computational Complexity

| Step | Complexity | Typical Time (T=60k) |
|------|-----------|----------------------|
| Key Selection | O(n_q · T · d_k) | ~3s |
| NNLS (per head) | O(t² · n_q) | ~2s total |
| Value Refit (per head) | O(t² · d_v + t³) | ~2s total |
| **Total (HighestAttn)** | **O(n_q · T · d_k + n_head · t² · n_q)** | **~7s** |

The Q_ref generation step (repeat-prefill) adds ~8s. Using the cheap K-proxy instead reduces this to <1s.
